# Security Policy

## Supported Versions

We release patches for security vulnerabilities. Which versions are eligible for receiving such patches depends on the CVSS v3.0 Rating:

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability, please do the following:

1. **Do not** open a public GitHub issue
2. Email the maintainer directly at: abukhashabehmohammad@gmail.com
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

We will respond within 48 hours and work with you to address the issue before making it public.

## Security Best Practices

When using this package:

- Always use the latest version
- Review the code if using in security-critical applications
- Report any security concerns immediately

## Acknowledgments

We thank security researchers and users who report vulnerabilities responsibly.
